# JSON Tools - 需求文档

## 1. 项目概述

### 1.1 背景

日常开发中频繁需要处理 JSON 数据的格式化、校验、转义等操作，现有在线工具依赖网络、存在隐私风险且体验参差不齐。需要一个轻量、离线、跨平台的本地桌面工具。

### 1.2 目标

- 构建一个**极低内存占用**的跨平台（Windows / macOS）桌面应用
- 提供 JSON 格式化/校验 和 转义/去除转义 两大核心功能
- 完全离线运行，无需网络连接
- 安装包小巧，启动迅速

### 1.3 目标用户

- 后端开发工程师
- 前端开发工程师
- 测试工程师
- 其他需要处理 JSON 数据的技术人员

---

## 2. 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | **Tauri 2.x** | Rust 后端，内存占用极低（~10MB），安装包小（~3MB），原生支持 Windows/macOS/Linux |
| 前端框架 | **原生 HTML/CSS/JS** 或轻量框架 | 无需构建工具链，减少依赖，保持轻量 |
| JSON 解析 | 前端原生 `JSON.parse` / `JSON.stringify` | 浏览器原生实现，性能最优，无需额外依赖 |

### 2.1 为什么选 Tauri 而非 Electron

| 对比项 | Tauri | Electron |
|--------|-------|----------|
| 安装包大小 | ~3-5 MB | ~50-80 MB |
| 运行时内存 | ~10-20 MB | ~80-200 MB |
| 启动速度 | 快 | 较慢 |
| 系统集成 | Rust 原生 | Node.js |

## 3. 功能需求

### 3.1 JSON 格式化与校验

**描述**：将压缩或格式混乱的 JSON 字符串格式化为可读的标准缩进格式，并校验其合法性。

**功能点**：

- [x] 输入 JSON 字符串，点击格式化按钮后输出缩进整齐的 JSON
- [x] 支持自定义缩进空格数（2 / 4 / Tab）
- [x] 校验 JSON 合法性，非法时高亮显示错误位置及错误信息
- [x] 支持大文件处理（至少 5MB JSON 文件不卡顿）
- [x] 一键复制格式化结果

**输入**：
```json
{"name":"json-tools","version":"1.0.0","description":"A lightweight JSON tool"}
```

**输出**：
```json
{
  "name": "json-tools",
  "version": "1.0.0",
  "description": "A lightweight JSON tool"
}
```

### 3.2 JSON 压缩

**描述**：将格式化的 JSON 压缩为单行，去除所有空白字符。

**功能点**：

- [x] 一键将格式化 JSON 压缩为单行
- [x] 压缩前自动校验，非法 JSON 给出提示
- [x] 显示压缩前后的大小对比

### 3.3 JSON 转义与去除转义

**描述**：对 JSON 字符串进行转义（添加转义字符）或去除转义（还原原始 JSON）。

**功能点**：

**转义**：
- [x] 将普通 JSON 字符串转为转义后的字符串（用于嵌入代码、日志等场景）
- [x] 处理 `"`、`\n`、`\t`、`\r`、`\\` 等转义字符

**去除转义**：
- [x] 将转义后的 JSON 字符串还原为正常 JSON
- [x] 自动识别多层转义并逐层去除
- [x] 去除转义后自动校验结果合法性

**转义示例**：

输入：
```json
{"key": "value with \"quotes\""}
```

输出：
```
{\"key\": \"value with \\\"quotes\\\"\"}
```

**去除转义示例**：

输入：
```
{\"name\": \"json-tools\", \"version\": \"1.0.0\"}
```

输出：
```json
{"name": "json-tools", "version": "1.0.0"}
```

### 3.4 辅助功能

- [x] 支持从剪贴板粘贴输入
- [x] 支持拖拽文件导入 JSON
- [x] 输入区域支持语法高亮
- [x] 深色/浅色主题切换
- [x] 键盘快捷键支持（Ctrl+Enter 格式化，Ctrl+Shift+E 转义等）

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 应用启动时间 | < 1 秒 |
| 内存占用（空闲） | < 20 MB |
| 内存占用（处理 5MB JSON） | < 100 MB |
| 格式化 1MB JSON | < 500ms |
| 安装包大小 | < 10 MB |

### 4.2 兼容性

- Windows 10 / 11（x64）
- macOS 12+（Intel & Apple Silicon）

### 4.3 离线运行

- 应用启动和所有功能完全离线可用
- 不依赖任何在线服务或 CDN
- 前端资源全部打包在应用内

### 4.4 安全性

- 所有数据处理在本地完成，不上传任何数据
- 不记录用户输入的 JSON 数据
- 不使用遥测或数据收集

## 5. UI/UX 设计要求

### 5.1 整体风格

- 简洁实用，无多余装饰
- 类似代码编辑器的视觉风格
- 左右分栏布局：左侧输入，右侧输出

### 5.2 布局设计

```
+--------------------------------------------------+
|  [Logo] JSON Tools          [主题切换] [关于]      |
+--------------------------------------------------+
|  [格式化] [压缩] [转义] [去除转义]  | 快捷键提示   |
+--------------------------------------------------+
|                    |                               |
|    输入区域         |       输出区域                |
|   (文本编辑器)      |     (只读/可复制)             |
|                    |                               |
|                    |                               |
+--------------------------------------------------+
|  字符数: xxx  |  大小: xxx KB  |  状态: 合法/非法   |
+--------------------------------------------------+
```

### 5.3 交互要求

- 操作按钮放置在顶部工具栏，一目了然
- 输入区域支持等宽字体显示
- 错误信息在输出区域醒目展示（红色高亮）
- 底部状态栏实时显示输入内容的基本信息
- 所有操作均有即时反馈，无模态等待

### 5.4 主题

- 浅色主题：白色背景，深色文字
- 深色主题：深灰背景，浅色文字，护眼舒适
- 默认跟随系统主题设置
